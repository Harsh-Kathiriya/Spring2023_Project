Glossary


  1. Provider - Health care professional that provides services to ChocAn members


  2. Manager - ChocAn employee that can view ChocAn data reports and verifies fees to be paid to ChocAn providers


  3. Operator - A person that interacts with the ChocAn data center to add, delete, or update member records


  4. Terminal - Device that the provider interacts with to communicate with the ChocAn Data Center


  5. Service Record - A record of a service provided, created by the data center to be sent to ChocAn for billing. Includes date, provider and member number, and a code corresponding to the type of service provided.


  6. Member Report - Record for individual members that includes the member name, number, street address, city, state, and zip code. For each service provided, it includes the date of service, provider name, and service name.


  7. Provider directory- A directory that is accessed when providers want to verify that the correct service was provided. This includes the current date and time, the date service was provided, provider number, member number, service code, and comments.


  8. ChocAn data center- Data center stores all data related to ChocAn services and is accessed to create reports of a variety of ChocAn data.


  9. Provider Report - A report was given to each provider at the end of the week detailing all the services he or she has provided to ChocAn members during that week.


  10. EFT Record - Record created by the data center of funds to be transferred from ChocAn to its various providers.


  11. Summary Report - A report given to the manager lists every provider to be paid that week, the number of consultations each had, and his or her total fee for that week.


  12. Member Record - Record of all information associated with an individual member using ChocAnâ€™s services. Can be updated during the day by operators.


  13. Accounting Process - Weekly process of reading all ChocAn services provided in that week and printing reports of each one.


  14. Acme - Third-party organization responsible for financial procedures such as recording payments of membership fees, suspending members whose fees are overdue, and reinstating suspended members who have now paid what is owed.


  15. Chocoholics Anonymous - An organization dedicated to helping people addicted to chocolate. Members pay a monthly fee to ChocAn in exchange for access to addiction help services.

Use Case Descriptions


Use Case: Summary report request

Context: A report that is sent to a manager for accounts payable.

Actors: Manager, Accounting Process

Main Success Scenarrio:

1. The system terminal displays a summary request for the manager.

2. The manager selects the summary report option.

3. The system goes through ChocAn Data Center database to pull information.

3.1. The system retrieves all the providers that are to be paid that week.

3.2. TThe system retrieves the number of consultations from members that week.

3.3. The system retrieves the total number of providers who provided services.

3.4. The system retrieves the total number of consultations.

3.5. The system retrieves the total overall fee for that week.

4. The system creates a file entailing all of the data that was retrieved in step 3.

4.1. A chart is created in the file.

4.2. The first column has the information in step 2.1.

4.3. The second column has the information from step 2.2.

4.4. The third column has the information from step 2.3.

4.5. The fourth column has the information from step 2.4.

4.6. The fifth column has the information from step 2.5.

5. The system prints the file.

Extensions:

1a. There is an error pulling the request.

1a.1. The system informs the manager.

1a.2. The use case continues at step 1.

2a. There is an error in retrieving information.

2a.1. The system informs the manager.

2a.2. The use case continues at step 1.

3a. The system cannot create a summary report file.

3a.1. The system informs the manager.

3a.2. The use case continues at step 1.

4a. The system cannot print the file.

4a.1. The system saves all of the information.

4a.2. The system informs the manager.

4a.3. The use case continues at step 1.
